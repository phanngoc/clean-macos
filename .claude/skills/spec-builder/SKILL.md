# Spec Generator Skill

Generate a complete specification suite that transforms ideas into actionable implementation plans.

## Purpose

This skill helps you create professional, comprehensive specification documents that include:

1. **Requirements** (requirements.md) - User stories with acceptance criteria
2. **Technical Design** (design.md) - Architecture, components, and interactions
3. **Task Breakdown** (tasks.md) - Granular, actionable implementation tasks

Perfect for product teams, developers, and AI coding agents to understand requirements, system architecture, and execution steps.

## ğŸ“‹ Phase 1: REQUIREMENTS Generation

### Objective
Create clear, testable requirements using user stories and acceptance criteria.

### Instructions

1. **Identify the Core User Story**
   - Write in format: "As a [user], I want to [action], so that [benefit]"
   - Focus on user value, not implementation

2. **Define Acceptance Criteria using EARS Format**
   - Use WHEN/THEN structure for clarity
   - Make criteria specific and testable
   - Cover both success and failure scenarios

3. **Specify Constraints and Boundaries**
   - Input requirements and validation rules
   - Output expectations
   - Performance requirements
   - Security considerations
   - Error handling requirements

## âœ… Phase 3: TASK BREAKDOWN Generation

### Objective
Break down the design into granular, actionable tasks suitable for implementation.

### Instructions

1. **Create Sequential Tasks**
   - Order tasks by dependencies
   - Each task should be completable independently once dependencies are met
   - Aim for tasks that take 2-8 hours

2. **For Each Task Include**:
   - Clear description of what needs to be done
   - Expected output/deliverable
   - Input requirements
   - Dependencies on other tasks
   - Acceptance criteria for completion
   - Complexity/time estimate (optional)

3. **Organize by Categories**:
   - Data/Schema tasks
   - Backend/API tasks
   - Frontend/UI tasks
   - Integration tasks
   - Testing tasks
   - Documentation tasks

4. **After Generating tasks.md**:
   - Extract tasks to CSV for tracking: `python track_progress.py extract <tasks.md>`
   - Use `python track_progress.py next` to get the next task to implement
   - Use `python track_progress.py implement <task_id>` to start working on a specific task
   - Mark tasks as done: `python track_progress.py done <task_id>`

## ğŸ“Š Task Progress Tracking

After generating `tasks.md`, you can use the built-in tracking system to monitor implementation progress.

### Setup Tracking

1. **Extract tasks from tasks.md to CSV**:
```bash
cd .claude/skills/spec-builder/scripts
python track_progress.py extract ../../specs/[feature-name]/tasks.md
```

This will parse all tasks from `tasks.md` and save them to `tasks.csv` with their metadata (category, complexity, dependencies, etc.).

### Track Progress

**View overall progress**:
```bash
python track_progress.py progress
```

**List all tasks**:
```bash
python track_progress.py list
```

**List tasks by status**:
```bash
python track_progress.py list --status=pending
python track_progress.py list --status=in_progress
python track_progress.py list --status=done
```

**List tasks by category**:
```bash
python track_progress.py list --category=Backend
```

### Mark Tasks

**Mark a task as done**:
```bash
python track_progress.py done <task_id>
```

**Update task status**:
```bash
python track_progress.py status <task_id> <status>
# Statuses: pending, in_progress, done, blocked
```

### Get Task for Implementation

**Get next available task** (respects dependencies):
```bash
python track_progress.py next
```

**Get specific task for implementation**:
```bash
python track_progress.py implement <task_id>
```

This command will:
- Mark the task as `in_progress`
- Display full task details
- Show the task's dependencies and requirements

### Workflow Example

```bash
# 1. Extract tasks from newly generated tasks.md
python track_progress.py extract ../../specs/my-feature/tasks.md

# 2. Check overall progress
python track_progress.py progress

# 3. Get next task to implement
python track_progress.py next

# 4. After implementing, mark as done
python track_progress.py done 5

# 5. Check progress again
python track_progress.py progress
```

### CSV Structure

The `tasks.csv` file contains:
- `task_id`: Task number from tasks.md
- `title`: Task title
- `category`: Task category (Backend, Frontend, Testing, etc.)
- `status`: Current status (pending, in_progress, done, blocked)
- `complexity`: Task complexity (Low, Medium, High)
- `estimated_time`: Time estimate
- `dependencies`: Comma-separated list of dependent task IDs
- `spec_path`: Path to the tasks.md file
- `updated_at`: Last update timestamp
- `notes`: Optional notes about the task

### Integration with Implementation

When implementing a task:

1. **Get the task**:
   ```bash
   python track_progress.py implement <task_id>
   ```

2. **Read the full task details** from `tasks.md` using the task_id

3. **Implement the task** following the acceptance criteria

4. **Mark as done**:
   ```bash
   python track_progress.py done <task_id>
   ```

The tracking system automatically:
- Preserves task status when re-extracting from tasks.md
- Respects task dependencies when suggesting next tasks
- Tracks progress by category and overall completion
